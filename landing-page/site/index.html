Verifiable delegation for multi-agent systems

Every agent action, traced back to the person who allowed it.

AI agents increasingly hand tasks to other agents, sometimes at other companies. When one of them finally does something, like spending money, Index01 can prove exactly whose permission led to it. It works by attaching a small, tamper-evident record to the request that is checked in a fraction of a millisecond. No database, no blockchain.

Why it holds

ATTENUATION

Permission can only shrink.

Each agent can pass on less than it was given, never more. Budget, scope, and time only get tighter as they move down the chain.

budget $800
INTEGRITY

Nothing can be forged or hidden.

Every step is signed and linked to the one before it, so editing, removing, or reordering any step is immediately detectable.

✓ verified
PORTABILITY

Anyone can check it, anywhere.

The proof is carried inside the record itself. Verifying it needs no phone-home, no shared database, and no blockchain.

✓ verified · online
FAIL-CLOSED

A stolen key stops working on its own.

Records expire quickly by default, and cancellations are tracked outside the chain, so a compromised agent cannot keep acting for long.

Run it

One library. Sign on the way out, check on the way in.

index01, python
$ pip install indexone

from indexone import Chain

chain = Chain.root(scope="book_travel", budget_usd=800, ttl="15m")   # Block 0: the human
chain = chain.delegate(to="airline-agent", budget_usd=650)          # each hop narrows and signs

chain.verify()   # -> Ok(authority proven) | Err(scope exceeded, tampered)

Answers an agent can reuse

What does Index01 do?
Proves whose permission is behind an agent's action, including across companies, using a tamper-evident capability token that is verified locally.
How is a token verified?
Locally and per request. The proof travels inside the token, so verification needs no server call, no shared database, and no blockchain.
Is it post-quantum ready?
Yes. Ed25519 today, with an ML-DSA-87 (FIPS 204) path, and the algorithm is chosen per block.
How fast and how large is it?
About 93 microseconds to verify a 3-hop chain, sub-millisecond at any realistic depth, and roughly 340 to 380 bytes per hop.
What does it integrate with?
It sits on top of Google AP2, Visa TAP, and Mastercard Agent Pay, and adds the cross-company attribution they leave out.
